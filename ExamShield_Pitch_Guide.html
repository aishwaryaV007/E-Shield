1. The 30-Second Elevator Pitch

The Pitch Concept

Every semester, academic institutions manually grade hundreds of thousands of handwritten scripts. This manual chain of trust suffers from transcription errors, summing slips, and student collusion networks that are invisible to a grader checking pages sequentially. ExamShield is a local computer vision assistant for exam evaluation cells. It reads digit scoring blocks, validates cover page additions, maps student collusion graphs via prose semantic analysis, and flags borderline scores for double checking—all offline on standard CPUs, maintaining total data privacy.

4. Core Evaluation Engines (Deep Dive)

🔢 MarkSafe Engine Trust Layer
PaddleOCR Digits + OpenCV Pixel Density
  • Extracts question marks and cover page totals from predefined templates.
  • Parses complex tokens using regular expressions, handling integers, decimals, fractions (e.g., 9/10), and evaluable additions (e.g., 4+3.5).
  • Safe Ambiguity Handling: Skipped and low-confidence markings (OCR confidence < 0.85) are marked AMBIGUOUS_MARK and routed to a reviewer instead of making false guesses.
  • Triggers SUM_MISMATCH if question sums fail to match the written overall total.
✍️ CopyCatch Engine Headline Demo
Prose OCR + sentence-transformers (all-MiniLM-L6-v2)
  • Converts handwritten answer prose to vector embeddings locally.
  • Class-Baseline Anomaly Ranking: Discards class-wide noise (common blackboard formulas, definitions) by computing z-scores: $Z_{A,B} = \frac{\text{Similarity}(A,B) - \mu_{\text{class}}}{\sigma_{\text{class}}}$.
  • Links collusion paths only if $Z_{A,B} \geq 3.0$ (representing standard deviations away from class average).
  • Generates interactive PyVis HTML node graphs. Graders click graph edges to inspect side-by-side student handwriting.
👤 ScriptID Engine Roster Auditor
Digit OCR + pandas Registration Lookup
  • Processes the student Roll Number zone on answer booklet covers.
  • Cross-references entries against class register database tables in CSV formats.
  • Flags UNREGISTERED_ID (OCR adjustments are easy via visual cover crops) and DUPLICATE_ID (if multiple booklets share a roll number).
  • Flags ABSENTEE_WITH_SCRIPT to protect institutions from exam-hall envelope logging leaks.
📈 ReEval Guard Engine Dispute Reducer
Pure Python Logic Overlay
  • Scans final scores against institutional grade cutoffs and pass-fail lines.
  • Identifies scripts located within border margins (e.g. 1-2 points below passing thresholds).
  • Adds scripts to high-priority review pipelines to resolve math checks prior to final score locking.
📐 BlankCheck & RubricLens (Stretch Features) Assist Engines
OpenCV Density + cross-encoder/nli-deberta-v3-xsmall
  • BlankCheck: Runs pixel density checks to mark blank pages, preserving physical workload history.
  • RubricLens: Uses Natural Language Inference to check paragraphs against rubrics. Highlight outputs show Entailment (Green) or Contradiction (Red). It never awards grades—the human decides.

5. Technical Stack & Local-Execution Justifications

Designed for air-gapped exam rooms, our stack avoids external dependencies and runs efficiently on standard laptop hardware.

Local OCR Engine PaddleOCR (CNN-based) Offline, free, and yields character confidence levels—vital for checking scribbled or crossed-out scores.

7. Q&A Defense Playbook (Judge Prep)

Q: OCR on handwriting is noisy. How do you avoid printing incorrect grades?
A: We operate under a strict human-in-the-loop policy. The system never inputs final marks automatically. If PaddleOCR confidence score drops below 85% due to a strikeout or scribble, MarkSafe locks it as AMBIGUOUS and prompts visual cover-page card reviews.
Q: Pairwise text comparison is $O(N^2)$. How does this scale for large batches?
A: Text is digitized and vectorized once. Cosine similarity comparisons over 1,000 papers ($10^6$ operations) on a standard CPU require less than 1.5 seconds using optimized NumPy matrix calculations, making scaling highly manageable.
Q: What if two students write similar answers because they memorized the same lecture slide?
A: This is why we implemented Class-Baseline Anomaly Ranking. If the whole cohort shares similar wording, the baseline mean ($\mu_{\text{class}}$) rises. CopyCatch maps z-scores; links only light up if student pairs diverge significantly from the class-wide average.
Q: Why not use commercial cloud models like GPT-4o?
A: Examination data is highly confidential and cannot be sent to third-party servers. Additionally, local CNNs (PaddleOCR) and sentence vectors (MiniLM) are completely free, run offline in secure cells, and supply character coordinates and confidence boundaries that cloud models hide.
