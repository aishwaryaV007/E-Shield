TRACK 02 // PREDICTIVE ANALYTICS (ML / DL)

🛡️ ExamShield — AI Handwritten Answer-Sheet Evaluator

"Train ExamShield on a few previously corrected answer sheets, and it grades new handwritten scripts the way your teachers do — marks, feedback, and a percentage, in minutes."

1. The 30-Second Pitch

Universities manually evaluate hundreds of thousands of handwritten answer booklets each semester — slow, inconsistent, and subjective. Different evaluators award different marks for the same answer, and nothing captures how teachers grade so it can be reused.

ExamShield learns marking behaviour from previously corrected scripts and then automatically evaluates new scripts:

  • Phase 1 — Train: on historical answers + teacher-awarded marks + answer keys + rubrics, we train a mark-predictor model.
  • Phase 2 — Evaluate: OCR a script → detect each question → measure semantic similarity to the answer key → the trained model assigns percentage-based marks → feedback + deduction reasons → an evaluated sheet.
100% local and offline. The mark is the trained model's prediction — never an LLM's (Track 02 prohibits LLM-generated predictions).

3. Two-Phase Architecture

PHASE 1 — TRAINING Historical corrected scripts + teacher marks + answer keys + rubrics → dataset_builder → features → trainer (XGBoost, tuned) → evaluate → mark_predictor.pkl + metrics (RMSE / MAE / R² / ±1-mark accuracy) PHASE 2 — EVALUATION Scanned scripts → ingestion (deskew/binarize) → handwriting OCR (+confidence) → segment questions (match answer key + rubric) → semantic similarity + concept coverage → scorer (trained model → marks, % bands) → feedback + deduction reasons → evaluated sheet (question-wise marks, total, %)

5. Tech Stack (offline, CPU, no LLM in the grading path)

  • Backend: Python 3.11 · FastAPI · OpenCV / pypdfium2 (image prep).
  • Handwriting OCR: TrOCR handwritten / PaddleOCR (local, + confidence).
  • Similarity: sentence-transformers all-MiniLM-L6-v2 (~80 MB).
  • Mark-predictor: XGBoost regressor (+ joblib); NumPy / pandas / scikit-learn / rapidfuzz.
  • Storage / UI: SQLite + JSON · Next.js 14 + React + TypeScript + Recharts.

7. Q&A Defense Playbook

Q: Isn't this just prompting GPT to grade? That's banned in Track 02.
No — the mark is an XGBoost regressor trained on teacher marks. We show its RMSE and ±1-mark accuracy on held-out data. No LLM is in the grading path.
Q: Handwriting OCR is noisy — how do you trust the marks?
Scoring uses semantic similarity (meaning, not exact characters). Low-confidence answers are flagged for human verification, never guessed.
Q: Differently-worded correct answers?
Sentence embeddings, not keyword matching, so they still score well; NLI catches negation so contradictions aren't counted as coverage.
Q: No historical teacher-marked data?
Fallback to an unsupervised similarity-to-key baseline; the trained model is the upgrade once labels exist.
Q: How is it measurable?
RMSE, MAE, R², and accuracy within ±1 mark of the teachers' marks on a held-out split.
